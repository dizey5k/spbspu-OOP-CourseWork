# Курсовая работа: Агрегатор данных из REST API

_Программная инженерия, 2 курс_  
_Тема: Агрегация данных из нескольких открытых REST API с сохранением в JSON/CSV и веб-интерфейсом_  
Год выполнения: **2025**  
**Текущее состояние:** Реализованы бэкенд (REST API) и фронтенд (React). Проект готов к демонстрации первого этапа.

## Описание проекта

Приложение позволяет:

- Получать данные из нескольких открытых REST API:
  - [Cat Facts Breeds](https://catfact.ninja/breeds) – породы кошек.
  - [JSONPlaceholder Users](https://jsonplaceholder.typicode.com/users) – тестовые пользователи.
- Сохранять агрегированные данные в файлы формата **JSON** или **CSV**.
- Выполнять HTTP-запросы к источникам параллельно (реализовано в бэкенде с контролем количества одновременных задач).
- Просматривать сохранённые записи через веб-интерфейс с фильтрацией по источнику.

Проект состоит из двух частей:

- **backend** – Spring Boot приложение, предоставляющее REST API для управления сбором данных и файлами.
- **frontend** – React + Vite приложение, реализующее пользовательский интерфейс.

### Основные компоненты backend

- **`ApiSource`** – интерфейс для добавления новых источников данных.
- Реализации: `CatFactsBreedsSource`, `JsonPlaceholderUsersSource`.
- **`AggregationService`** – сбор данных из выбранных источников.
- **`FileService`** – сохранение и загрузка файлов в формате JSON/CSV.
- REST контроллер `ApiController` с эндпоинтами:
  - `GET /api/sources` – список доступных источников.
  - `POST /api/fetch` – запуск сбора данных и сохранение в файл.
  - `GET /api/data` – чтение сохранённых записей из файла.

### Основные компоненты frontend

- Страницы: «Сбор данных» и «Просмотр данных».
- Компоненты: `SourcesList` (выбор источников), `FetchForm`, `DataViewer`, `RecordCard`.
- Адаптивный дизайн с поддержкой тёмной темы.
